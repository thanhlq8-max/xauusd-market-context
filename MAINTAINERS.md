## Review policy

- Changes must identify the affected module and user impact.
- Runtime changes require focused tests and the full release gate.
- Documentation and examples must preserve the monitor-only contract.
- External source adapters require documented rights and validation behavior.
- Pine logic, thresholds, and score weights are not accepted into the external repository without a separately locked specification.

## Community expectations

Issues and pull requests are reviewed according to maintainer availability. No response-time promise is made. Security reports must use `SECURITY.md` rather than a public issue.
